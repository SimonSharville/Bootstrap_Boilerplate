Megamenu

I have created 2 variations of megamenu. A fixed layout version (default) and a multi-column version.

Megamenus, like navbars have various nested versions depending on how deep the page is in the file structure. So, index.html has -n0 appended while any pages in the UI folder have -n1, as the pages are nested one folder deep. The reason for this is that using CodeKit to import .kit files into html pages saves time and coding but the links within the import get broken. I can't find away around this other than to make various versions of the same file with slighly different links.(../link.html etc). It's complicated, but less so than having to add links to every page. If I add a new file, I just have to add them to the navbar or megamenu .kit pages.

Megamenu Default version

The deafult version uses the standard Bootstrap columns to arrange the lists. The list can be laid out exactly as you wish them to appear.

Megamenu multi-column version

This version uses a single Bootstrap column with the classes multi-column multi-column-4 (or 2 or 3) added. This spreads the links over the colums evenly and can save space. However. You can't set a header to be always at the top of a column.

Note: this will not work in IE 9 and below.
