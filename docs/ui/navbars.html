Navbar Default

I have created this as one of two versions for use as a default. It contains a standard megamenu.

Note: Due to the way CodeKit imports .kit files I have had to create separate navbars for pages that sit in the root folder and those that sit nested within other folders. ie. The top level is named navbar-default-n0.kit (not nested), The next is named navbar-default-n1.kit for pages nested 1 deep, others will need to be made for deeper nested pages if required. The links within these change slightly, it's a pain but being able to import files means you only need to change the links in a few navbars, rather than on all pages.

Megamenues

Megamenues also have nested variations, but not the footer as this has no local links.

There are also 2 types of Megamenu. This is the standard version, with all the links within separate columns. There is another version, which I am using as default on this boilerplate that uses multi-columns to arrange the links.

See Megamenu page for more info.

Navbar Admin

This is the navbar-admin-n1 (which also has nested versions). It is the same as above but has the colouring for the admin section.

Second navbar

A second navbar can be added directly below the top navbar and given either the default or admin colours. This should have a class of navbar-second added to it.

The second navbar can be split into 2 columns of various widths as shown below. ie. those below use 8 & 4 and 6 & 6.
